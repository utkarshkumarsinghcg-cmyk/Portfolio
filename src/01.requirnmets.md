# **Portfolio Requirements (React-Based)**

## **1. Overview**

Students are required to build a **personal portfolio website using React.js** that showcases their skills, projects, and achievements in a professional manner.

The portfolio will be evaluated based on:

* Technical implementation
* Completeness of content
* SEO practices
* Responsiveness
* Design and overall presentation

---

## **2. Technology Requirements**

* The portfolio **must be built using React.js**
* Use of **React Router** is recommended (optional but preferred)
* The application must be **fully responsive** (mobile-first approach)
* Deployment can be done on any platform (Netlify, Vercel, GitHub Pages, etc.)

---

## **3. Domain & Identity Requirements**

* The **domain name must include your own name**
* Portfolio must clearly display:

  * Your **full name**
  * Your **profile photo** (in hero section)

---

## **4. SEO & Indexing Requirements**

The portfolio must follow proper SEO practices:

* Proper configuration in `index.html`:

  * `<title>` must include your name
  * `<meta description>` must describe you
  * Relevant meta tags for SEO

* Must include:

  * `robots.txt` file
  * `sitemap.xml` file

* Website must be submitted and verified on:

  * Google Search Console

### For Refrance: https://github.com/codinggita/CGxSU_Semester_1/tree/main/react(sem_02)/07.SEO
---

## **5. Mandatory Sections in Portfolio**

The portfolio must contain the following sections:

### **5.1 Navigation Bar**

* Clean and accessible navbar
* Easy navigation across all sections/pages

---

### **5.2 Hero Section**

* Your **name**
* Your **photo**
* Short professional introduction

---

### **5.3 Skills Section**

* Technical skills (Frontend, Backend, Tools, etc.)
* Use visual representation (progress bars, cards, icons)

---

### **5.4 Projects Section**

Each project must include:

* Project title and description
* Your **GitHub profile link**
* Project live link
* **YouTube demo video**
* If Full Stack:

  * API documentation (from Postman and published)
* If applicable:

  * Figma design link

---

### **5.5 Project Categories (Mandatory Subsections)**

Projects must be categorized into:

* Games created by you
* Clones (e.g., Netflix clone, Amazon clone)
* Full Stack applications
* Frontend projects

---

### **5.6 Figma Designs Section**

* Showcase UI/UX designs created using Figma
* Include preview images and links

---

### **5.7 Certificates Section**

* Display all certifications with:

  * Title
  * Issuing organization
  * Proof (image or link)

---

### **5.8 Hackathon Section**

* Hackathons participated in
* Problem statements
* Solutions and outcomes

---

### **5.9 Achievements Section**

* Awards, recognitions, milestones

---

### **5.10 Resume Section**

* Resume must be displayed via:

  * View button (PDF viewer / modal)
* **Auto-download is strictly prohibited**

---

### **5.11 Contact Section**

* Contact form using:

  * EmailJS or similar
* Should allow users to send messages directly from frontend

---

### **5.12 Social Media Section**

Must include all of the following:

* LeetCode
* GitHub
* LinkedIn
* Twitter (X)
* YouTube Channel

(All links must be working and clearly visible)

---

## **6. Additional Functional Requirements**

* Proper routing (if implemented)
* Smooth scrolling and transitions
* Clean UI/UX
* Fast loading performance

---

## **7. Design Requirements**

* Modern and professional UI
* Consistent color scheme and typography
* Proper spacing and layout
* Optional:

  * 3D elements
  * Animations
  * Interactive components

---

## **8. Evaluation Criteria**

The portfolio will be evaluated based on:

* Completeness of all required sections
* Technical implementation (React, responsiveness)
* SEO implementation
* Project presentation quality
* Design and creativity
* Functionality (links, forms, navigation)

---

## **9. Submission Checklist**

Before submission, ensure:

* Portfolio is deployed and accessible
* Domain includes your name
* All links are working
* SEO files (`robots.txt`, `sitemap.xml`) are present
* Website is submitted to Google Search Console
* Resume is viewable (not auto-downloaded)

---

## **10. Evaluation & Scoring**

* The portfolio will be evaluated out of a **total of 40 marks**

### **Marks Distribution**

* **Structure & Mandatory Sections – 10 Marks**

  * All required sections present (navbar, hero, skills, projects, certificates, hackathon, achievements, contact, social links, resume)
  * Proper organization and accessibility

* **Projects Showcase Quality – 10 Marks**

  * GitHub, live links, and YouTube demos properly added
  * Full-stack projects include Postman documentation
  * Proper categorization (games, clones, frontend, full stack)
  * Figma links where applicable

* **SEO & Deployment – 5 Marks**

  * Proper meta tags in `index.html`
  * robots.txt and sitemap.xml present
  * Google Search Console integration
  * Domain includes your name

* **Responsiveness & Functionality – 5 Marks**

  * Fully responsive design
  * Smooth navigation and working links
  * Functional contact form (EmailJS or similar)

* **Design & Presentation – 10 Marks**

  * Clean and modern UI
  * Proper use of components
  * Unique design and layout
  * Additional features (themes, animations, 3D, interactions)

---

### **Important Evaluation Rules**

* Showcasing the **right components properly** will increase scores

* **Additional features** (themes, unique UI, advanced interactions) can help achieve maximum marks

* **Complete and meaningful content is mandatory**

* **No fake content or images will be accepted**

* Certificates must include:

  * Proper description
  * Valid proof

* Hackathon entries must include:

  * Supporting materials (project repo, demo video, etc.)
